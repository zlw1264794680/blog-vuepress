# 搭建win开发环境

搭建开发环境 (opens new window)

必须安装的依赖有：Node、JDK 和 Android Studio、python（官网没说，但是是要安装的）

  • node14
  • jdk11
  • python27

Android Studio安装步骤，按照官网步骤执行即可。特别需要注意的是：sdk版本、avd设备的api版本！！！

# 初始化项目

npx react-native init AwesomeProject

可能会出现拉取失败报错，即使翻墙了，也会失败。如下操作，指定npm安装，默认yarn

npx react-native init YourAwesomeProject --npm

Error while creating react-native cli project (opens new window)

# 启动项目

npm run android

如果报错的话，先排除依赖包问题，删除项目下的 node_modules 目录 和 package-lock.json ,重新 npm install 解决问题。

npm cache clean --force

npm install

# 错误：app：installDebug

任务'：app：installDebug'的执行失败反应本机应用程序[重复] (opens new window)报错的解决方法：

  • sdk版本 要与android/build.gradle该文件下的buildToolsVersion设置的版本保持一致或者比它版本高，向下兼容。
  • sdk版本 对应的 avd设备的api版本，是否兼容？
  • 排除以上俩个问题，可能是 没有找到模拟器、模拟器未开机成功、手机开发者模式没有打开、运行了多个模拟器 等情况造成，执行npm run start，通过按键a重新部署到模拟器，直到构建成功！！！

# 错误：app停留在启动页

# 问题描述：

app构建成功，但是会停留在启动页，即程序并未真正启动！！！

# 导致问题的原因

启动（夜神）模拟器 or 真机后（android的avd设备不需要启动），执行如下命令，完成app的构建：

npm run android

只执行这一步就可能会导致，app构建成功，但未启动。

# 解决方法

npm run start

# 按 r ，刷新app，重新构建即可
r

# 分析原因：

摘自官网 (opens new window)

编译并运行 React Native 应用，确保你先运行了模拟器或者连接了真机，然后在你的项目目录中运行 yarn android 或者 yarn react-native run-android。

此命令会对项目的原生部分进行编译，同时在另外一个命令行中启动 Metro 服务对 js 代码进行实时打包处理（类似 webpack ）。 Metro 服务也可以使用 yarn start 命令单独启动。

所以，当没有 js 代码的时候，该app能正常启动。当存在 js 代码的时候，需要再执行 yarn start。

yarn android

yarn start
