0.1+0.2不等于0.3

zlw
2023-03-12
  • JS
  • 二进制

# 原理分析

0.1+0.2不等于0.3的原因，在于编程语言所有计算的底层逻辑都是0101的二进制数据。

Js语言也不例外，至于为什么会是0.30000000000000004？跟Js语言以及现代浏览器存储二进制的方式有关。

  • 在JS内部所有的计算都是以二进制方式计算的。
  • 在现代浏览器中是用浮点数形式的二进制来存储二进制。

# 计算过程

具体计算过程，这里没有展开讲解，具体的讲解，已经是对二进制的学习了，可以参考：0.1+0.2 为什么不等于0.3_0.1+0.2不等于0.3 (opens new window)

  • 十进制转成二进制
    • 0.1和0.2转成的二进制是无穷的。
  • 二进制转成二进制浮点数
    • 在现代浏览器中是用浮点数形式的二进制来存储二进制
  • 浮点数二进制进行相加减

# 小结

0.1+0.2 不等于 0.3 ，因为在 0.1+0.2 的计算过程中发生了两次精度丢失。

  • 在 0.1 和 0.2 转成双精度二进制浮点数时，由于二进制浮点数的小数位只能存储52位，导致小数点后第53位的数要进行为1则进1为0则舍去的操作，从而造成一次精度丢失。
  • 在 0.1 和 0.2 转成二进制浮点数后，二进制浮点数相加的过程中，小数位相加导致小数位多出了一位，又要让第53位的数进行为1则进1为0则舍去的操作，又造成一次精度丢失。
  • 最终导致 0.1+0.2 不等于0.3 。
