# React Native 与 Flutter

# 介绍

React Native 是一个由 Facebook 开发的框架，用于构建跨平台移动应用程序。它使用 JavaScript 和 React 库来开发 IOS 和 Android 应用程序， 并且具有大量的第三方库和工具，使得开发人员可以轻松地构建高质量的应用程序。

Flutter 是 Google 开发的一个移动应用程序开发框架，它使用 Dart 编程语言。Flutter 具有独特的“一切皆为小部件”的设计思想，使得开发人员可以快速地构建漂亮、流畅、高度定制化的应用程序。

# 比较

  • 性能：Flutter 在性能方面表现更出色，因为它使用自己的渲染引擎和编译器，并可以在不同平台上实现高性能和流畅度。
  • 开发体验：React Native 更容易上手，因为它使用 JavaScript 编程语言和 React 库。Flutter 需要学习新的语言（Dart）和框架。
  • 第三方库：React Native 拥有更多的第三方库和工具，这意味着开发人员可以更容易地找到所需的功能并加速开发过程。
  • 生态系统：React Native 拥有更广泛的生态系统，包括社区、教程、博客等等。Flutter 的生态系统相对较新。
  • 文档：React Native 的文档相对完善，并且有很多社区支持。Flutter 的文档相对较少，并且需要更多时间去了解。

总之，选择 React Native 还是 Flutter 取决于您的需求和项目要求。如果您想要快速开始并且寻找广泛的支持和第三方库， 则 React Native 是一个不错的选择。 如果您需要最佳性能、流畅度和高度定制化，则 Flutter 可能更适合您。

# 搭建开发环境

React Native的环境搭建 (opens new window)

mac系统

brew install node
brew install watchman

node 可以使用 nvm 来进行安装管理。

Watchman 则是由 Facebook 提供的监视文件系统变更的工具。安装此工具可以提高开发时的性能（packager 可以快速捕捉文件的变化从而实现实时刷新）。
